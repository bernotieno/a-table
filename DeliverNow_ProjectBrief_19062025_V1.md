# DeliverNow — Full Redesign of the Delivery Ecosystem  

## Project Name  
**DeliverNow — Full Redesign of the Delivery Ecosystem (Clients, Restaurants, Runners, Runner Managers)**  

## Background  
DeliverNow partners with restaurants to offer take-away delivery using a network of runners.  

Recently, DeliverNow received negative feedback from customers, runners, managers, and restaurant owners:
- ❌ Customers: Inaccurate delivery times, too much or unwanted cutlery/condiments  
- ❌ Runners: Unfair or inefficient route allocation  
- ❌ Managers: Lack of visibility and control over runners  
- ❌ Restaurants: Difficulty managing on-site and take-away preparation  

DeliverNow seeks to redesign its full digital experience to address these challenges and improve satisfaction across all user types.  

## Design Goals  
- 🎯 Deliver accurate, real-time delivery estimates to clients  
- 🎯 Enable clients to customize delivery (e.g. no cutlery, no condiments)  
- 🎯 Provide runners with fair, optimized routes + clear task flows  
- 🎯 Equip managers with effective runner tracking + task allocation tools  
- 🎯 Help restaurants manage in-house + take-away orders smoothly  

## Target Users  
### 👤 Client / Customer  
- Orders food for delivery or pick-up  
- Uses mobile + desktop  
- Wants fast, accurate delivery with clear customization options  

### 👤 Restaurant Owner / Staff  
- Prepares orders for dine-in + take-away  
- Uses desktop + mobile  
- Needs clear view of incoming orders, prep timing, delivery handover  

### 👤 Runner / Delivery Person  
- Delivers food from restaurant to client  
- Uses mobile  
- Needs clear route, task list, live updates  

### 👤 Runner Manager  
- Oversees delivery team  
- Uses desktop + mobile  
- Needs fleet dashboard, task reallocation, runner monitoring  

## Problems to Solve  
- ✅ Inaccurate delivery estimates frustrate clients  
- ✅ Extra/unwanted condiments and cutlery create waste  
- ✅ Runners receive inefficient or unfair routes  
- ✅ Managers lack visibility and flexible tools for fleet management  
- ✅ Restaurants struggle to coordinate dine-in and take-away prep  

## Opportunities  
- 💡 Real-time order tracking for clients  
- 💡 Delivery customization (cutlery, condiments, notes)  
- 💡 Dynamic route optimization for runners  
- 💡 Manager dashboard for fleet visibility + task management  
- 💡 Restaurant order management system with separate workflows for dine-in and take-away  

## Scope  
- Design mobile + desktop interfaces for:  
  - Clients  
  - Restaurant owners / staff  
  - Runners  
  - Runner managers  
- Build one full user journey prototype per persona (order flow)  
- Mid-fidelity prototype + testing (Week 3)  
- High-fidelity prototype + testing (Week 5)  

## Constraints  
- Timeline: 5 weeks  
- Team: Max 3 people  
- Must test mid-fi and hi-fi prototypes with at least 5 users  
- Design for both mobile and desktop where applicable  

## Success Metrics  
- 📈 Fewer delivery complaints (wrong timing, waste)  
- 📈 Increased client satisfaction in post-order surveys  
- 📈 Runners report higher satisfaction with routes  
- 📈 Managers report better control over fleet  
- 📈 Restaurants report easier order handling  

## Design Process  
✅ **Empathize:** Research + interviews (real users)  
✅ **Define:** Problem statements + journey maps  
✅ **Ideate:** Brainstorm + sketch wireframes  
✅ **Prototype:** Mid-fi and hi-fi clickable prototypes  
✅ **Test:** User tests at both stages
